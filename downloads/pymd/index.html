pymd: wrapper for Python Markdown

Posted
2013-11-17
Source
Get source

Hey!

pymd is now available in PyPI. That means that you can install it like pip install pymd :)

pymd is a command line wrapper for Python Markdown library but having a full HTML file instead of only what it's originally parsed.

Features

  • Batch process files (in folder or in a .list) with .md, .txt or .markdown extensions
  • Specify an output folder (with automatic creation)
  • Includes all the extensions installed by Python Markdown and you can even specify your own!
  • Merge files into one big HTML or create a little book with navigation links. You can even create your own index!
  • CSSed: leave the no-so-ugly embeded CSS (you can even switch between sans and serif fonts) or link to one of your own if you prefer.
  • Headers for files! So you can share the same metadata (like your beautiful name and the creative title of your work) in all the files or when you merge the files and forgot to include the main title of your project.
  • Wiki links: [](file|path) -> [title linked file](output_path)
  • Made with love <3

Requirements

  • Python (> 2.7)
  • Python Markdown
  • pymd !

Usage and examples

The syntax is the following:

pymd.py SOURCE [--output FOLDER [--flat]] [--header FILE ] [--exts LIST ] [ --css FILE | --serif ] [ book [--index FILE --nav] | merge [--toc(0, 1, 2, 3, 4, 5)] ]

But don't be afraid, I'll explain you with examples.

To parse a file in its folder, just run:

pymd.py nicefile.md

Note

source can be a file, a folder or a .list which contains the file paths.

Process files in a folder and save the resulting ones in another, creating a book (with "prev" and "nav" links) using the embebed CSS with serif font:

pymd.py /srcfolder book -o /dstfolder --serif

Note

-o is the same as --output and book is the same as --book. You can also use -n o --nav to use the titles of the files as navigation links.

The index.html is created automatically, which links to the files using their title.

Hint

If you don't like the default index.html, you can create your own linking to the resulting files with markdown links. You can use "wiki style" links

Merge many files in a big HTML using a .list, linking to a .css and using a header:

pymd.py nice.list merge -o /dstfolder --header im_a_header.md --css prettify.css

Note

merge is the same as --merge

The nice.list only contains files path to process. Linking to a .css (existent or not) will look like <link href="prettify.css" rel="stylesheet" type="text/css" /> in the HTML.

Hint

If you use the merge option, you can order the files as you want in a .list (and don't rename files that can't be renamed), so they will be parsed and merged in that order.

im_a_header.md could contain the main title, author, date, comments, table of contents, etc. In this case, is the first, but you could include it in each file (without using merge, of course).

Note

--header is only necessary if the filename is other than _header

A header example would be:

Title: Word domination project Summary: Ideas for the word domination project Comment: Missing budget list

[TOC_HERE]

If you want to change the maximum depth of the TOC, you can do it with --toc. By default, all the levels are shown.

Only the first level:

pymd.py nice.list -m -o /dstfolder --toc 1

Show up to the second level:

pymd.py nice.list -m -o /dstfolder --toc 2

Lastly, if you've installed Python Markdown extensions and want to use them, just include them in the command separated by commas:

pymd.py nicefile.md --extensions del_ins,grid_tables
