pandy es un wrapper para pandoc + cositas copadas (si usas markdown -> html)

Basicamente toma un archivo/carpeta, de qué convertir y a qué convertir, y lo corre por pandoc.

Uso

Para usar el script, solo llamalo con algo asi:

pandy source format_from format/s_to [other options]

Explicación por ahi abajo.

Caracteristicas

  • Sintaxis un porquito mas facil para los comandos de pandoc
  • Carpeta de output manteniendo o no la estructura
  • La fuente puede ser un archivo, una carpeta o un .list que contenga las rutas a los archivos
  • Formatos: algunos se puede abreviar (mirar abajo)
  • Formato de salida: se pueden poner más de uno! solo separalo con espacios.
  • Podes crear un librito re lindo (html, solo añade navegación entre archivos). Si no te gusta/queres el "Next-Prev", podes usar los titulos de archivo. Crea un lindo y simple index o incluí el tuyo.
  • Usa un archivo de configuración: así no tenes que acordarte de todos los argumentos o escribir un "wrapper" para un script "wrapper" :)
  • Soporte para [TOCME]. Significa que el archivo que lo tenga, va a tener TOC. No mas eso de separar archivos con TOC, los que no tienen y finalmente ponerlos en una carpeta.
  • Si usas markdown y convertis a HTML hay cositas lindas para vos: abbreviations (de las verdaderas, no como esas que son ignoradas por pandoc) y admonitions.
  • "WikiLinks" para tu index!

Explicación

De un archivo/carpeta/.list, poner de qué formato se va y a cual se convierte, que puede ser una lista separada por espacios. Los formatos son una selección de los mas usados:

  • from: docbook, html, json, latex, markdown, markdown_github, markdown_mmd, markdown_phpextra, markdown_strict, mediawiki (o mw), opml, rst, textile
  • output: los de arriba + asciidoc, beamer, docx (o doc), epub, epub3, fb2, html5, odt, opendocument (o opendoc), pdf, plain, rtf, slides (o slide)

Nota

  • Todos los "markdown"s pueden escribirse como "md": markdown -> md; markdown_github -> md_github; etc.
  • El formato de slides se especifica con --slides
  • Extensiones de markdown automaticamente habilitadas: link_attributes, hard_line_breaks

Se pueden usar algunas opciones de pandoc pero con otros nombres:

--output, -o Carpeta de salida --self archivo contenido --header FILE Archivo Header. Incluido como es (Incluido como es: en crudo, verbatim) --footer FILE Archivo Footer. Incluido como es (Incluido como es: en crudo, verbatim) --index FILE Index personalizado para book. Puede usar wikiLinks --html4 Usar html4 en lugar de html5 --merge, -m Unir archivos. --slides Formato para slides --bib FILE Archivo de bibliografía --css FILE CSS externo --highlight Stilo de highlight --highlight-no Sin highlight --tpl FILE Archivo de template --toc, -t incluir TOC --depth Profundidad de TOC --hide Esconder e-mail --sections Envolver en , identificadores ahi en lugar de los titulos --pandoc PANDOC Path de Pandoc. Default: pandoc --data-dir FOLDER Carpeta de datos

Y un par de mios:

--flat No mantener la estructura de la carpeta --book, -b Crear un libro con navegación (next/prev) e index --nav, -n (PAra book) usar titulos en la navigación --config FILE Usar archivo de configuración (option=valor)

Si usas markdown y convertís a HTML, hay regalitos para vos. Podés tener abbreviations, como en PHP Markdown Extra:

Some text with an ABBR and a REF. Ignore REFERENCE and ref.

Y admonitions:

<admon "class/type" "optional title">
* markdown
* super
* content
</admon>

El "class/type" puede ser algo como "information", "danger", "tip", "hint", "hungry", "duck", "yay" ...

También podes incluir una etiqueta para TOC ([TOCME]) para que ese archivo tenga TOC en lugar de poner --toc o separar archivos en dos grupos y después ponerlos juntos. (Lo añade automaticamente después de buscar el archivo, nada de magia)

Para book: podes crear tu propio index y usar "wikiLinks" como [](file|nice_file.txt). Se convierte a [title of file](nice_file.html)

Si no te gusta poner las opciones en el CLI, o tener un script que llame a pandy con los argumentos que quieras, podes crear un archivo de configuración con key=value (como los ini). Ejemplo:

PANDOC_DATA_DIR = "C:\Program Files\Pandoc" TEMPLATE = 'github.html' HIGHLIGHT= 'zenburn' FORMATS_TO = [html, pdf] NAV = True

Especifica el archivo con --config (la extensión no importa, los INI headers son ignorados asi como los comentarios (line o inline). No te preocupes)

Y si, baja pandy de Github :)
