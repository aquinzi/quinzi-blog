pymd: wrapper para Python Markdown

Publicado
2013-11-17
Source
Obtener código

Hey!

pymd ahora esta en PyPI. Eso significa que lo podes instalar como pip install pymd :)

pymd es un wrapper para la librería Python Markdown, basicamente te permite tener un HTML completo a comparación de lo que te tira la librería.

Características

  • Proceso en batch (en una carpeta o usando .list) con extensiones .md, .txt o .markdown
  • Especificar una carpeta de resultados/output (con creación automática)
  • Incluye todas las extensiones de Python Markdown y hasta podes especificar las tuyas!
  • Combinar los archivos en un HTML grande o crear un librito con links de navegación; hasta podes crear tu propio index!
  • CSSed: trae un no-tan-feo CSS embebido (hasta podes cambiar entre tipografía sans y serif) o linkear a una que prefieras.
  • Encabezados para los archivos! Así podes compartir el mismo metadata (como tu hermoso nombre y tu creativo título) en todos los archivos o cuando combinas y te olvidaste poner un título principal.
  • Wiki links: [](file|path) -> [title linked file](output_path)
  • Hecho con amor <3

Requerimientos

  • Python (> 2.7)
  • Python Markdown
  • pymd !

Usos y ejemplos

La sintaxis es la siguiente:

pymd.py SOURCE [--output FOLDER [--flat]] [--header FILE ] [--exts LIST ] [ --css FILE | --serif ] [ book [--index FILE --nav] | merge [--toc(0, 1, 2, 3, 4, 5)] ]

Pero a no temer, acá te explico con ejemplos.

Para convertir un archivo en su propia carpeta, solo basta con correr:

pymd.py nicefile.md

Nota

El source puede ser un archivo, una carpeta o un .list que contendrá las rutas de los archivos.

Convertir los archivos en una carpeta y pasarlos a otra, creando un librito (con links "prev" y "nav") usando el CSS embebido con tipografía serif:

pymd.py /srcfolder book -o /dstfolder --serif

Nota

-o es lo mismo que --output y book es lo mismo que --book. También podes usar -n o --nav para tener los links de navegación con los títulos de los archivos

Te crea el index.html automaticamente, donde linkea a los archivos usando su título.

Hint

Si no te gusta el index.html creado, podes hacer el tuyo linkeado a los archivos finales con links normales de markdown. Podes usar links "estilo wiki"

Unir varios archivos en un gran HTML usando un .list, linkeando un .css y usando un header:

pymd.py nice.list merge -o /dstfolder --header im_a_header.md --css prettify.css

Nota

merge es lo mismo que --merge

El nice.list solo contiene rutas a los archivos a procesar. Incluyendo un .css (existente o no) se verá como <link href="prettify.css" rel="stylesheet" type="text/css" /> en el HTML.

Hint

Si usas la opción merge, podes aprovechar y ordenar los archivos como vos quieras en un .list (y no renombrar archivos que no se puedan). Así se convertirán y combinarán en ese orden

El im_a_header.md podria contener el título principal, el autor, la fecha, comentarios, la tabla de contenidos, etc. En este caso, es lo primero que se muestra, pero también se podria incluir en cada uno de los archivos (sin usar el merge, por supuesto)

Nota

--header solo es necesario si el archivo de encabezado tiene otro nombre que _header

Un ejemplo de header podria ser:

Title: Word domination project Summary: Ideas for the word domination project Comment: Missing budget list

[TOC_HERE]

Si queres cambiar la profundidad máxima del TOC, podes hacerlo con --toc. Por default, se muestran todos los niveles.

Para mostrar el primer nivel:

pymd.py nice.list -m -o /dstfolder --toc 1

Y para mostrar los dos primeros niveles:

pymd.py nice.list -m -o /dstfolder --toc 2

Por último, si instalaste otras extensiones para Python Markdown y las queres usar, solo incluilas en el comando separados por comas:

pymd.py nicefile.md --extensions del_ins,grid_tables
