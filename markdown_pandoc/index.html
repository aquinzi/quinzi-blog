Basic

Same as Markdown.

Paragraphs separated by one or more blank lines. If you want to separate/insert a <br> tag, the line must end with two or more spaces, then type return.

Emphasis: * and _. If you surround an * or _ with spaces, it'll be treated as a literal asterisk or underscore. To produce a literal asterisk or underscore, you can backslash escape it.

  • Italic: <em>. One * or _
  • Bold: <strong>. Double * or _

Headers: Setext and atx

  • Setext-style headers: "underlined" using equal signs (first-level headers) and dashes (second-level headers). Any number of underlining ='s or -'s will work.

    This is an H1
=============

This is an H2
-------------

  • Atx-style headers: 1-6 hash characters at the start of the line, corresponding to header levels. You may "close" atx-style headers but closing hashes don't even need to match the number of hashes used to open.

    # This is an H1

## This is an H2

###### This is an H6

Blockquotes:

  • email-style >
  • hard-wrapping text with > before every line

  • before the first line of a hard-wrapped paragraph:

    > This is a blockquote with two paragraphs. Lorem ipsum dolor sit amet,
consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

> Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse
id sem consectetuer libero luctus adipiscing.

  • Nested by adding additional levels of >:

    > This is the first level of quoting.
>
> > This is nested blockquote.
>
> Back to the first level.

  • Can contain other elements:

    > ## This is a header.
> 
> 1.   This is the first list item.
> 2.   This is the second list item.
> 
> Here's some example code:
> 
>     return shell_exec("echo $input | $markdown_script");

Lists: Markers may be indented by up to three spaces, must be followed by one or more spaces or a tab. List items may consist of multiple paragraphs and can include other elements

  • unordered (bulleted): asterisks, pluses, and hyphens -- interchangably -- as list markers:

    *   Red
+   Green
-   Blue

  • ordered (numbered): use numbers followed by periods

    1.  Bird
2.  McHale
3.  Parish

    The actual numbers you use have no effect on the output. It's possible to trigger an ordered list by accident (writing a number-period-space sequence at the beginning of a line), to avoid this, backslash-escape the period.

Code Blocks: wrapped as <pre> and <code> tags (both). Indent every line of the block by at least 4 spaces or 1 tab. A code block continues until it reaches a line that is not indented (or the end of file). Within a code block, & and <, > are automatically converted into HTML entities. Markdown syntax is not processed within code blocks.

This input:

This is a normal paragraph:

    This is a code block.

Generates:

<p>This is a normal paragraph:</p>

<pre><code>This is a code block.
</code></pre>

Horizontal Rules: three or more -, *, or _ on a line by themselves. You may use spaces in between:

* * *

***

*****

- - -

------------------------

Links: The link text is delimited by [square brackets]

  • Inline: after link text [], follow (). Inside the parentheses, put the URL to point to, along with an optional title for the link, surrounded in quotes.

    This is [an example](http://example.com/ "Title") inline link.

[This link](http://example.net/) has no title attribute.

  • Reference-style: use a second set of square brackets, inside which you place a label to identify the link. Link definition names may consist of letters, numbers, spaces, and punctuation, they are case insensitive. Link definitions can be placed anywhere.

    This is [an example][id] reference-style link.

    Then, anywhere in the document, you define your link label like this:

    [id]: http://example.com/  "Optional Title Here"

    The following three link definitions are equivalent:

    [foo]: http://example.com/  "Optional Title Here"
[foo]: http://example.com/  'Optional Title Here'
[foo]: http://example.com/  (Optional Title Here)

    The link URL may, optionally, be surrounded by angle brackets:

    [id]: <http://example.com/>  "Optional Title Here"

    You can put the title attribute on the next line and use extra spaces or tabs for padding:

    [id]: http://example.com/longish/path/to/resource/here
    "Optional Title Here"

    The implicit link name shortcut allows you to omit the name of the link (the link text itself is used as the name). Just use an empty set of square brackets.

    [Google][]

    And then define the link:

    [Google]: http://google.com/

Code: wrap it with backtick quotes (`). To include a literal backtick character, you can use multiple backticks as the opening and closing delimiters. Ampersands and angle brackets are encoded as HTML entities automatically.

``There is a literal backtick (`) here.``

Images: resembles links syntax.

  • Inline:

    ![Alt text](/path/to/img.jpg)

![Alt text](/path/to/img.jpg "Optional title")

  • Reference-style:

    ![Alt text][id]

    Where "id" is the name of a defined image reference. Image references are defined using syntax identical to link references:

    [id]: url/to/image  "Optional title attribute"

Automatic Links: surround the URL or email address with angle brackets.

Backslash Escapes:

\   backslash
`   backtick
*   asterisk
_   underscore
{}  curly braces
[]  square brackets
()  parentheses
#   hash mark
+   plus sign
-   minus sign (hyphen)
.   dot
!   exclamation mark

Pandoc

Extra Basic formatting

  • Deleted text: This is deleted text.

    This ~~is deleted text.~~

  • Super & sub script: H2O is a liquid. 210 is 1024.

    H~2~O is a liquid.  2^10^ is 1024.

Lists

Ordered & Unordered

Pandoc allows starting with the number:

  1. Ninth
  2. Tenth
  3. Eleventh
    1. subone
    2. subtwo
    3. subthree
 9)  Ninth
10)  Tenth
11)  Eleventh
       i. subone
      ii. subtwo
     iii. subthree

Pandoc will start a new list each time a different type of list marker is used. So, the following will create three lists:

(2) Two
(5) Three
1.  Four
*   Five

outputs:

<ol start="2" style="list-style-type: decimal">
    <li>Two</li>
    <li>Three</li>
</ol>
<ol style="list-style-type: decimal">
    <li>Four</li>
</ol>
<ul>
    <li>Five</li>
</ul>

If default list markers are desired, use #.:

  1. one
  2. two
  3. three
#.  one
#.  two
#.  three

Attributes

Headers may have id:

A setext style header

A hash style header

A setext style header {#setext}
=================================

### A hash style header ### {#hash }

The above results in the following output:

<h1 id="setext">A setext style header</h1>
<h3 id="hash">A hash style header</h3>

To link to another section of the same document, use the automatically generated identifier. For example:

See the Introduction or See the Introduction.

See the [Introduction](#setext) or See the [Introduction].
